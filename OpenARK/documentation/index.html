SCSS Structure

This template uses SCSS for stylesheets. For more information about SCSS, visit http://sass-lang.com/guide.

Template SCSS files are located in the /SCSS directory. Vendor CSS files are located in the /html/css directory.

We need to compile SCSS files to CSS to make them usable. All compiled files are located in the /css folder.

All SCSS files are separated by component types for easy customization. There is a main file for the theme style.scss file which has references to all .scss files that compose the theme. This files uses Bootstrap-Sass variables and mixins in order to compile to CSS.

You can modify those files to include additional style or remove any unwanted reference, one use case could be to leave the variation of a section you like and remove references to the others.

Do not remove references to Bootstrap components as it will not compile.

We include a basic but functional GULP script in order to compile the SCSS to CSS, as well as a copy of Bootstrap-Sass component.

To compile open a terminal window and type: gulp sass. Make sure you have installed the dependencies in package.json file before, by running npm install command.

(function() {
    var gulp = require('gulp');
    var sass = require('gulp-sass');
    var autoprefixer = require('gulp-autoprefixer');

    gulp.task('sass', function () {
        return gulp
            .src('./sass/*.scss')
            .pipe(sass({
                includePaths: ['./bootstrap-sass/assets/stylesheets/']
            }))
            .pipe(autoprefixer({
                browsers: ['last 2 version', '> 5%']
            }))
            .pipe(gulp.dest('./css'));
    });
})();

Once you have run the task, all .scss files will appear in the CSS folder as respective CSS code compiled.

Scripts

Javascript

This template uses several JS files. Those files are loaded at the bottom of the page to increase the site speed and rendering.

Some of the javascript files used are:

  • bootstrap.js - Bootstrap framework functions.

Please refer to Credits section to see the complete list of js files.

Server side

We use some PHP scripts to process the submission of the forms on the template

  • srv - Contains all components we use to process the requests
    • components - Contains the core components we use
    • integration - Contains the main class to interact with MailChimp API
    • models - We use OOP in php, here are the models of each form in the template
    • contact - Contains the logic for the submission of Contact Form
    • subscribe - Contains the logic for the submission of Subscribe Form
    • trial - Contains the logic for the submission of Trial Form

Every form in the HTML has an action attribute with the path relative to the PHP file which will process it

<form id="form-contact" action="srv/contact.php" data-msg="section">
    ...
</form>

By default we left it this way, you can move the files to another location or rename them as you like, if you do that don't forget to update the action attribute to reflect your changes. Please refer to the forms section for additional details on the file structure and PHP code

MailChimp

We use MailChimp API 3.0 to connect to the service, our implementation only include Adding a member to a list, for this sake we create a class in /srv/integration/MailChimp.php file.

We implement this class to make the subscription process in subscribe.php. There you must provide API_KEY and LIST_ID with your own. We assume you know how to get those values.

define("LIST_ID", "[YOUR LIST ID]");
define("API_KEY", "[YOUR API KEY]");

Fonts

We use Google Source Sans Pro and Lato fonts by default.

To change the embedded font, open the HTML page you want and locate the head section and change the font as you wish. Make sure to change the font names in CSS files as well.

<link href="https://fonts.googleapis.com/css?family=Lato:100,300,400,700" rel="stylesheet">
<link href="https://fonts.googleapis.com/css?family=Source+Sans+Pro:300,400,700" rel="stylesheet">

Once you have changed the font, look for $font-family-template and $font-family-complementary variable in /SCSS/_variables.scss and adjust properly.

$font-family-template: "Source Sans Pro", $font-family-base;
$font-family-complementary: "Lato", $font-family-template;

Color scheme

This template is ready to use any colors scheme, for wich we use SCSS to create colors pallete and process the CSS dynamically and show your colors right away.

You change your colors scheme in /SCSS/_variables.scss.

We use 6 main colors for each scheme which can be completely customized to fit your needs.

To include your own colors just change variables to your values and compile.

$color-1: #ffffff !default;
$color-2: #4E5A70 !default;
$color-3: #1B99B9 !default;
$color-4: #18B295 !default;
$color-5: #181c1f !default;
$color-6: #e5e8ea !default;

Video background

We use Vide to include full screen video background, please refer to its documentation to know more about it.

As a brief explanation, you just include data-vide-bg attribute with the path to the video as the value without the extension, video and poster must have the same name.

<header id="main-header" class="header-site header-video" data-vide-bg="video/up" data-vide-options="posterType: jpg">
...
</header>

This way you need to have a file named up.mp4|up.ogv|up.webm in the video folder, as well as up.jpg file which is the image cover

Overlay, padding & blurred

Overlay

In addition to the image cover background, each section may include an overlay. There are so many colors of overlays as defined in the colors scheme, all named by its index (.overlay-1, .overlay-2, .overlay-3 and so on). You can choose the level of opacity 10 to 90 as well as one of dotted or squared patterns overlay.

Make sure you have included .overlay class.

<div class="overlay overlay-4 alpha-9">
    <!-- CONTENT HERE -->
</div>

Padding

You can define how much space you will give to each section by configuring the padding of its container, we have set up three main classes for padding: half-padding, quarter-padding and full-padding this is the default value and is not necessary to include in your code. The sizes of the padding can be configured and are set in _variables.scss

$padding-default: $line-height-computed * 3; // ~60px
                    
<div class="overlay overlay-4 alpha-9">
    <div class="container quarter-padding">
        <!-- CONTENT HERE -->
    </div>
</div>

Blurred

You can define if include a nice blurred effect in your header hero image, just include blurred class to the header element, in order for this feature to work you must include the background image in the :before pseudo element

<header id="header-site" class="header-site header-preview blurred parallax">
    <!-- CONTENT HERE -->
</div>
                    
.header-preview {
    &:before {
        background-image: url("../img/header/bg/4.jpg");
    }
}

Forms

Next comes with fully functional form submission. Every form has an action attribute, this is the script which will process the form in server side. We have set up those scripts in srv folder.

<form action="srv/subscribe.php" class="form form-subscribe" method="post" data-msg="this">
...
</form>

Each script has references to the core libraries we have developed.

// srv/suscribe.php
include('./integration/MailChimp.php');
include('./models/SubscribeModel.php');
                    
// srv/models/SubscribeModel.php
namespace models;

include __DIR__ . '/../components/Model.php';
use components\Model;

All of these references are located relatively to the scripts. You can change the location of the files or rename them, by doing this you must update the files to reflect your changes.

Note how each file has include statements, and how the path is relative to its container, this way PHP knows where to find those references.

data-msg

Advice data-msg="this".

This attribute indicates where the notification message should appear when a response is available. Use any valid css selector, we'll prepend a message box container to it. We'll find the element by traversing up through the form ancestors in the DOM tree, this keyword indicates the message will be prepended to the form itself.
